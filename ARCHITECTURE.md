# Architecture Overview

This project provides a comprehensive framework for generating datasets, processing construction documents (PDFs and images), estimating costs using LLM agents, and evaluating their performance. The architecture is divided into three core functional areas: Dataset Generation, the Evaluation Pipeline, and the Evaluation Framework.

---

## 1. Dataset Generation
These components are responsible for creating both synthetic and real-world ground-truth datasets for cost estimation.

* **Dataset Builders (`setup_dataset.py`, `build_rich_dataset.py`)**: Scripts that compile the final dataset JSONs (`eval_dataset.json` and `rich_eval_dataset.json`) by combining images, generating spec texts, downloading real PDFs, and assembling cost breakdowns.
* **`cost_model.py`**: An RSMeans-based deterministic cost engine that generates accurate ground-truth costs across 48 building types, simulating real-world variances and locational factors.
* **`spec_generator.py`**: A synthetic text generator that creates realistic Construction Specifications Institute (CSI) specification texts for synthetic cases.
* **`rich_dataset_sources.py`**: A catalog and download manager for real-world LOC HABS architectural drawings and links to actual state/federal procurement PDFs.

---

## 2. Evaluation Pipeline (The "Solver")
This represents the agentic system being evaluated. It takes massive unstructured construction documents and floor plans, distilling them into a single cost estimate.

### Step A: Document Processing (`pdf_pipeline.py`)
A two-agent pipeline responsible for parsing and summarizing large specification textbooks.
* **Text Extraction & Truncation**: Uses PyMuPDF to extract text from dense specification PDFs. It includes a smart truncation strategy to heavily prioritize the start (approx. 80%) and the end (approx. 15%) of the text to fit within LLM context limits while capturing fundamental project details and technical divisions.
* **Agent 1 (Structured Field Extraction)**: Processes the text in overlapping chunks to extract definitive fields like building type, area, stories, and present CSI divisions.
* **Agent 2 (Qualitative Insights)**: Analyzes the chunks sequentially to identify cost drivers, complexity factors, risks, schedule constraints, site conditions, and material quality signals.
* **Guardrails**: Incorporates a final step (`enforce_json_length_limit`) to ensure the combined output JSON is strictly under 20,000 characters, proactively truncating long text fields to avoid ballooning context windows in the next step.

### Step B: Cost Estimation (`cost_estimator.py`)
The final reasoning agent that synthesizes structured data and visual data.
* **Inputs**: Consumes the JSON output from `pdf_pipeline.py` along with all architectural floor plans (encoded as Base64 images).
* **LLM Call**: Typically uses an external LLM via OpenRouter (e.g., Gemini 3 Flash). The model is prompted with a strict system instruction to output only a scalar numeric value representing the total cost.
* **Tools**: The LLM is explicitly equipped with code interpretation tools (if supported by the provider interface) to perform complex cost calculations on the data before returning the final number.

---

## 3. Evaluation Framework
* **`evaluate.py`**: The scoring mechanism that takes the output JSON generated by `cost_estimator.py` and compares it against the ground truth (`eval_dataset.json` or `rich_eval_dataset.json`).
* **Metrics**: Calculates Mean Absolute Percentage Error (MAPE), Mean Absolute Error (MAE), Root Mean Squared Error (RMSE), R-squared (R²), and exactness thresholds (e.g., % of estimates within 10%, 20%, 30% of actuals).
* **Configuration (`eval_config.yaml`)**: Stores global validation logic, acceptable percentage tolerances, acceptable MAPE ranges, and CSI division definitions for granular analysis.
